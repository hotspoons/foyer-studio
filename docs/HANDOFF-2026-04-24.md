## TL;DR

Eight features landed in this session. They're all committed to the
working tree but not yet pushed:

1. SPDX + LICENSE split (GPL-2.0-or-later for `shims/ardour/`, Apache-2.0 everywhere else)
2. Relay chat — FAB + panel, markdown + yaml/json highlighting, admin-clear, snapshot-to-disk
3. Push-to-talk — binary frames on the existing control WS (no extra socket), speaker banner + pulse ring
4. Track browser-source routing — host picks which peer sources audio per track; transport-bar mic button appears for assigned peers; live monitoring force-off
5. Tunnel-fallback fix — 1200 ms grace → 2500 ms + cancel on first greeting; the `force-fallback` mount race is gone
6. Session-restoration fix — `foyer-app` primes state from `store.state` at construction so it doesn't depend on having been mounted when envelopes fired
7. RBAC broadening — chat + PTT + `list_track_browser_sources` added to performer/viewer default allow-lists (bundled `defaults/roles.yaml`)
8. FAB rail defaults + tear-outs — Actions / Session / Windows are now tearable-out FABs; all six FABs (those three + Agent + Chat + Layouts) default-docked in the right rail via a new `defaultDocked` meta flag + `_dockSeen` tracking
9. Global-shortcut typing guard — Space / arrows / etc. no longer hijack text inputs inside shadow DOM (chat composer, agent input)

Plus infra refactor (partial):

- External deps convention: Ardour clones into `<repo>/ext/ardour`
  (gitignored) via `just ardour clone`. Legacy `/workspaces/ardour`
  sibling layout still honored as a fallback.
- Audio passthrough: Dockerfile adds `vscode` to `audio` group;
  devcontainer.json has a commented `--device=/dev/snd` +
  `--group-add=audio` block users uncomment on Linux hosts.
- Multi-root `.code-workspace` file deleted.

## Fresh-clone setup

```bash
# 1. Clone + open in devcontainer
git clone <foyer-studio-repo> foyer-studio
cd foyer-studio
code .                                     # "Reopen in Container"

# 2. Wait for post-create.sh + user-post-install.sh (installs bun,
#    node, playwright — once per container lifetime).

# 3. Stub-backend dev loop (no Ardour required)
just run --backend stub

# 4. Real-Ardour dev loop (slow one-time: ~15 min)
just ardour clone                          # into ext/ardour
just ardour build                          # waf configure + build
just run                                   # full Ardour backend

# 5. Verify
just ci                                    # fmt + clippy + tests + Playwright
```

## Architectural decisions that should get logged

These came up in-session and are worth a new `docs/DECISIONS.md` entry
each (see the ADR format in that file — they're short and numbered):

1. **Chat rides the control WS as binary frames** — not a dedicated
   `/ws/ptt` route. Reasoning: per-origin HTTP connection cap (6 on
   HTTP/1.1) is tight with `/ws` + `/ws/audio/master` already open;
   PTT bandwidth is trivial (~5 KB/s per speaker); mixed text+binary
   on one socket is a 10-line protocol addition vs. a whole new
   route. See the wire-format comment block in
   `crates/foyer-server/src/chat.rs`.

2. **`defaultDocked` FAB registration + `_dockSeen` set** — seeds a
   FAB into the right rail the first time it registers, but records
   any subsequent user dock/undock so reloads don't override. New
   localStorage key: `foyer.layout.fabSeen.v1`.

3. **Priming Lit component state from `store.state`** — post-three-
   tier-split, components are constructed AFTER bootstrap has
   already processed the greeting + initial envelopes. Components
   that only react to future events miss their own seed data. Fix
   pattern: read `store.state.*` at construction AND subscribe for
   updates. See `web/ui-full/app.js` constructor.

## Behavioral changes to be aware of

- **Right-dock starts closed by default.** Previous behavior was
  "open on first launch with the Actions panel visible." Now the
  rail is always visible but the panel only opens when the user
  clicks a specific FAB. Rationale: the three hardcoded panels
  (Actions, Session, Windows) became regular FABs, and forcing the
  panel open for one of them would be arbitrary.
- **"Open session" click from welcome screen now always re-fires
  `select_session` + `request_snapshot`** — previously the store's
  `setCurrentSession` early-returned when the id matched, which
  broke reload-then-click when the id was auto-focused by the
  session_list reducer.
- **Transport Space bar no longer captures inside text inputs.**
  Applies to chat composer, agent composer, and any future
  custom-element text entry. The check now uses `composedPath()` so
  it works through shadow roots.
